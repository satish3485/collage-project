ALOHANET

New requirements

In the late 60’s, several attempts were made to provide remote access to computer systems and information progressing systems through the existing worldwide telephone network, but soon it became clear that the network was shaped by the requirements of voice communications and not data networking. And it was there that Norman Abramson1 had the idea of using radio communications instead of the telephone system.

The development

In 1968, at the University of Hawaii, Norman Abramson leaded a group of research to develop ALOHANET2, a network which would have connected users from Oahu to all the other Hawaiian islands, and viceversa, all through low-cost equipment. The original version of ALOHA used two distinct frequencies, with the server machine broadcasting packets to everyone on the “outbound” channel, and the client machines sending data to the server on the “inbound” channel. If data was received correctly by the server, an “acknowledgment” packet was sent back to the client, and if the client didn’t receive it, it would have sent again its data, and so on. This mechanism was used to detect and correct conflicts that could occur when two client machines attempted to send data at the same time, and it was needed because unlike the ARPANET, where each node could only talk directly to a node at the other end of a wire or satellite circuit, in ALOHANET all client nodes communicated with the server on the same frequency.

The random-access channel

The Hawaiian Islands.

This solution reduced critically the complexity of the protocol and the networking hardware, and became the basis for subsequent Ethernet development and later Wi-Fi networks. ALOHANET used a random-access channel for communication between users, which was later called “pure ALOHA channel”: this approach gives automatically bandwidth to every user that has some data to send. In later versions of the system, some radio relays3 were placed in order to connect the main network on the island of Oahu to other islands.
