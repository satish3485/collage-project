Gopher

Gopher is a TCP/IP layer protocol designed for distributing, searching, and retrieving documents over the Internet. It was designed from the start to look and work like a read-only global network file system. It was made to work like a basic file system, so it was composed of a series of hierarchical hyperlinkable menus. The protocol uses the TCP port 70 to connect to a Gopher server.

You can't directly check out a Gopher site with today's web browser, unless you can use extensions or have an intermediate proxy server do the translation for modern browsers1.

Origins

Regarded as the predecessor and, in its early stages, an alternative to the World Wide Web, the first Gopher system was released in the spring of 1991 by a team led by Mark P. McCahill from the University of Minnesota, whose mascot is a gopher (Goldy Gopher), hence its name. It's also an appropriate name because a gopher digs the ground to reach a desired location and also represents an assistant who runs errands.2

The Golden Gopher

Its main goals were:3

  • Act as a distributed document delivery system
  • Familiar file-like hierarchical structure
  • Simple syntax
  • Cheap and quick system to create
  • Implementation of searches

Evolution

Growth

While the Web was still in its infancy, the higher education at the time saw the ease of setting up a Gopher server for Campus-Wide Information Systems (CWISs), adding to that links to other sites’ line directory and resources contributed to its quick adoption. In 1992, the standard for looking up an e-mail address was in the Gopherspace.

Originally, Gopher was text-only, but eventually evolved to graphical clients. It even got a new version called Gopher+, that allowed additional functions for the graphical clients. Network administrators loved it because it used few network resources and the first large-scale electronic network of libraries.

On November 17, 1992, the search tool Veronica (Very Easy Rodent-Oriented Net-wide Index to Computerized Archives) was introduced, providing a searchable index of Gopher. It was so popular that it became overloaded by the traffic. It is the equivalent of Archie for FTP.4

Decline

However, HTML quickly started dominating and people started migrating from gopher servers to the Web. A commonly held idea was spread around that the Gopher protocol almost disappeared because of the threat that, in 1993, the University of Minnesota would request fees for using the protocol, while HTML was completely open since its inception and its form was a lot less rigid.4

Because of that, the University of Minnesota released the source code of its applications tied to the protocol under the public GNU licence. Robert Alberti, co-author of the protocol, contested this opinion in his memoirs by demonstrating that its decline came from the introduction of the image tag (IMG) in HTML, which up until then only displayed text, like Gopher.5
