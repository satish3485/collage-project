UDP

UDP, which stands for "User Datagram Protocol", uses a simple connectionless transmission. It is used for applications that do not require the reliability of TCP, reducing latency and delivery validation. It also uses uses IP to actually get the data, for the network layer. UDP is the transport layer instead of TCP and does not verify the integrity of the data received, which means the application that uses UDP had to check that the data is the correct one. It is especially useful for small data exchanges, like chatting applications. Like TCP it provides port numbers to help distinguish different user requests and, optionally, a checksum capability to verify that the data arrived intact.



Origins

The protocol was designed by David P. Reed in 1980 and formally defined in RFC 768. The division of TCP, paved the way for a more efficient way of transmitting data, UDP. The goal was to make it simple and fast, so no checking of packets. Even the checksum, the computer checks if the data is intact, was optional. It doesn't even guaranty the arrival of the message, it is connexion-less, so it just sent packages and hopefully they arrive at destination.

Transmission process

  1. Higher-Layer Data: An application sends a message to the UDP software.
  2. UDP Message Encapsulation: The higher-layer message is encapsulated into the Data field of a UDP message. The headers of the UDP message are filled in, including the Source Port of the application that sent the data to UDP, and the Destination Port of the intended recipient. The checksum value may also be calculated.
  3. Transfer Message To IP: The UDP message is passed to IP for transmission.

Format

The format of a UPD header is a lot smaller than the TCP headers, that is only eight bytes against 20 bytes or more for TCP headers. A UDP header is divided in four part of 2 bytes each:

Source PortPort from which the message is to be sent
Destination PortPort to which the massage is to be sent to
LengthLength of the entire UDP datagram, so both header and the Data Field
ChecksumCan be optionally used to do a checksum
