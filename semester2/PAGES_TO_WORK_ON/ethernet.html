Ethernet

History

Ethernet is a computer networking technology used in Local Area Networks and created in 1973 by Robert Metcalfe while working as a researcher at the Xerox Corporation's Palo Alto Research Center(Xerox PARC). The idea for Ethernet was largely inspirated by ALOHAnet, another historically important computer network, that Metcalfe had studied as part of his PhD dissertation. He developed both the physical method for connecting the devices and the standards for monitoring the communication.

In 1979 Metcalfe co-founded the 3Com Corporation with the intention of commercializing Ethernet. 3Com started by building Ethernet circuit boards for minicomputers, but later it was developed into an Ethernet card, mostly known as NIC (Network Interface Card). He also succeeded convincing Intel, Xerox and DEC to work together and promote Ethernet as a standard for LAN networks.

On September 30, 1980 the three companies published "The Ethernet, A Local Area Network. Data Link Layer and Physical Layer Specifications" creating the earliest Ethernet standard. Version 2 was released on November 1992.

Ethernet battled against IBM's token ring and General Motor's token bus, two different LAN standards. At the end of the 1980s, 3Com shifted from coax cables to twisted pair cables and finally won the battle becoming the dominant local area network technology.

3Com was acquired by Hewlett-Packard on April 12, 2010. Since the acquisition, 3Com has been fully absorbed and no longer exists as a separate company.

Technology

Bus topology

The architecture of the original Ethernet used a bus topology where each computer is connected to a single cable. This implementation turned out in a broadcasting comunication, where all the messages are received by all the computers connected to the network.

There are several advantages using this kind of architecture. The network is very cheap and it's possible to connect a new device easily without modifying the entire system, a great plus for a network administrator. On the other hand there are also some problems, two computers could start sending informations through the cable at the same time resulting in a collision that inevitably compromises the messages.

The collision issue is solved by a scheme known as Carrier Sense Multiple Access with Collision Detection (CSMA/CD) which govern the way the computers shared the channel. First of all each station ensures that the line is free before starting to send data. If a collision occurs, both stations detect it and stop transmitting data, then they wait a random amount of time and try to transmit the data again. The collisions issue could start being a problem for high traffic networks.

The files are not delivered completeley in one step but split into several sections called frames. Each frame follows a specific format shown in the figure below. The file contains both the origin and destination address. Since each station receives all the frames delivered in the network, it will just check the destination address of them, keep the right frames and get rid of the others. This behaviour could be an issue since a station set in a promiscuous way could just sniff all the frames, even those frames which are not supposed to be delivered to it.

Star topology

The introduction of hubs and switches was a giant leap forward for Ethernet networks. A switch is a computer network device that connects stations together using a form of packet switching to deliver data to the right destination. This means that every station is only connected to the switch. As soon as the switch receives a frame it analyzes the destination address and passes the frame on to the channel connected to the right device. A hub is just a simpler version of the switch where every frame is redirected to all the lines instead of only the right one. The combination of switches and full-duplex connections eliminates the problem of collisions, since each Ethernet section is now isolated.

Evolution of Ethernet

During the years the physical layer components such as cables, connectors, etc. evolved, therefore improving the speed and reliability of Ethernet connections.
An Ethernet controller runs at the speed of the slowest connected device, which is helpful when mixing old and new technology on the same network.

Name Standard Clause Description
Experimental Ethernet No Standard The original Ethernet implementation had a bandwidth of 2.94 Mbit/s and used a coaxial cable.
Thick Ethernet 10BASE5 The first standard Ethernet used an AMP 228752-1 connector also known as a "vampire tap" that literally tapped a connection by drilling into the cable. It had a 10 MBit/s speed and used a bus topology with collision detection.
Fast Ethernet 100BASE-T A term that includes all the three standards for 100 Mbit/s over a twisted pair cable. All of these use a star topology.
Gigabit Ethernet 1000BASE-T Released in June 1999, all the variants and successive Ethernet versions use a star topology. The 1000BASE-X version use fiber optic cable while the other versions still use twisted pair cable.
10Gbits Ethernet 10GBASE-SR Released in 2002 it's ten times faster than the predecessor, most of the versions use fiber optic cable.
40Gbits Ethernet 40GBASE-KR4 Released in June 2010, is currently the latest version of Ethernet.
